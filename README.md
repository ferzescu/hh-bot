# hh_bot

Бот для автоматического поднятия резюме на hh.ru. Использует Playwright для эмуляции действий в браузере (публичный API hh.ru для соискателей закрыт с декабря 2025).

## Что делает

- Логинится на hh.ru
- Находит все резюме и нажимает «Поднять в поиске»
- Повторяет каждые 4 часа
- Логирует результаты в файл и stdout

## Быстрый старт

### 1. Настройка

```bash
cp .env.example .env
```

Заполнить `.env`:

```
HH_LOGIN=email@example.com
HH_PASSWORD=your_password
UPDATE_INTERVAL_HOURS=4
```

### 2. Запуск в Docker (рекомендуется)

```bash
docker-compose up -d
```

Просмотр логов:

```bash
docker-compose logs -f
```

Остановка:

```bash
docker-compose down
```

### 3. Запуск локально

```bash
pip install -r requirements.txt
playwright install chromium
python -m bot.main
```

## Первый запуск

При первом запуске бот залогинится через форму и сохранит cookies в `storage/cookies.json`. При последующих запусках cookies используются повторно.

Если на аккаунте включена двухфакторная аутентификация, бот запросит OTP-код в консоли. Для работы в Docker без ручного ввода рекомендуется отключить 2FA на hh.ru.

## Логи

Логи пишутся в `storage/hh_bot.log` (ротация: 5 MB, 3 файла) и в stdout.

Пример:

```
[2026-02-28 10:00:00] INFO: Сессия активна (cookies)
[2026-02-28 10:00:03] INFO: Резюме "Python Developer" — обновлено
[2026-02-28 10:00:03] INFO: Обновлено резюме: 1 из 1
[2026-02-28 10:00:03] INFO: Следующее обновление через 4 ч.
```

## Структура проекта

```

│   ├── browser.py     — поиск и клик кнопок обновления

│   └── logger.py      — настройка логирования
```

## Переменные окружения

| Переменная | Описание | По умолчанию |
|---|---|---|
| `HH_LOGIN` | Email или телефон на hh.ru | — |
| `HH_PASSWORD` | Пароль | — |
| `UPDATE_INTERVAL_HOURS` | Интервал обновления в часах | `4` |

## Troubleshooting

**Бот не находит кнопки обновления** — CSS-селекторы hh.ru могут измениться. Откройте hh.ru в браузере, найдите кнопку «Поднять в поиске» через DevTools и обновите `data-qa` атрибуты в `bot/browser.py`.

**Cookies перестали работать** — удалите `storage/cookies.json`, бот залогинится заново при следующем запуске.

**Ошибка авторизации** — проверьте логин/пароль в `.env`. Если hh.ru требует капчу, попробуйте сначала залогиниться вручную в обычном браузере.
